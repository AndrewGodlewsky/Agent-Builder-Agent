The silent oversize failure

Files over the ceiling still appear as SharePoint search hits but aren't processed by generative answers — it looks like the agent "found" the doc, yet nothing from it grounds the answer. Escape hatch: the 512 MB upload path. (research 03 · Learn: SharePoint no-response)

Ceiling, not target

500 sources / 500 files is a cap, not a goal. Top-3 grounding means volume dilutes rather than deepens — curated, well-named, well-described sources win. (research 04 §6)

Universal file rules

Everywhere: max 512 MB; no image/video/audio/executable files (images only work embedded in a PDF, with alt-text); no encrypted / sensitivity-labeled / password-protected files — they show "ready" but return nothing. (research 07 · Learn: file upload)

Design around, don't instruct around

Instructions can't modify retrieval, so none of these are fixable with prompting. If you need them, use a Dataverse table, a Power Automate flow, or a custom tool — not SharePoint knowledge. (research 03 §4–5)
