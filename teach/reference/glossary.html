How to use this: these are the exact senses in which the field manual uses each term. Where a definition carries a load-bearing number, the limits cheat-sheet has the citation; where it implies a decision, the decision flowcharts walk it.

A
Allow ungrounded responses
Agent setting that gates grounding. Off: the agent blocks any response generated in a turn where it didn't use a knowledge source or tool, and the fallback topic triggers. On: it may answer from general model knowledge.
B
Bing Custom Search
The retrieval backend for public-website knowledge sources; searches are restricted to your configured domains. Also exists as a classic-only source type usable inside a generative answers node.
C
Chunking
Splitting a file into pieces for indexing and retrieval. Chunk size and strategy are not configurable and not published by Microsoft — you cannot tune how documents are split.
Citations
The source references attached to grounded answers. PDFs added via SharePoint file sync get page-level citations; citations can't be piped into other tools or actions.
Classic orchestration
The older orchestration mode: knowledge search runs only as a fallback when no topic answers, and per-source input counts are far lower (e.g. 4 websites, 4 SharePoint sources, 2 Dataverse sources).
Copilot connectors (Graph / enterprise connectors)
Knowledge sources that surface enterprise systems (Jira, GitHub, CRM…) already indexed by Microsoft Search. Admin-configured, permission-trimmed, unlimited in generative mode.
Crawled property
Content and metadata extracted from an item during a SharePoint crawl. Raw material only — it must be mapped to a managed property before it enters the search index.
D
Dataverse
The Power Platform data store where uploaded knowledge files are copied, chunked, and vector-indexed per agent. Also the destination for the SharePoint file-upload path (Path B).
Dataverse search
Environment-level setting that must be enabled before file- or Dataverse-backed knowledge works at all. An admin action, and a common silent blocker at build time.
Dataverse tables (knowledge source)
Source type that queries structured business records with a retrieval-augmented technique built for tables. Unlimited sources in generative mode; classic caps at 2 sources × 15 tables.
Delegated permissions
The identity model for SharePoint retrieval: every query runs on behalf of the querying user's Microsoft Entra ID, which needs at least Read permission on the content.
E
Embedding (vector)
A numerical representation of content stored in multi-dimensional space where semantically similar items cluster together — the basis of semantic similarity search.
Enhanced search results
SharePoint-knowledge toggle that must be ON (with a same-tenant M365 Copilot license) to unlock the 200 MB ceiling and better retrieval. Left OFF, you are capped at 7 MB even when licensed.
F
Fallback (topic)
The system topic that fires when the agent can't answer — including every blocked turn when Allow ungrounded responses is off. Under classic orchestration, knowledge search itself ran only as a fallback.
Few-shot examples
Input→output example pairs written into instructions — Microsoft's recommended technique for edge cases. Never store examples as knowledge: retrieval is conditional and fragmentary (top-3 chunks).
File groups
Grouping multiple uploaded files behind a single knowledge source (one name + description), so routing contends with fewer, better-described sources — the recommended relief valve for the ~5-usable-sources ceiling.
G
Generative answers
The feature that summarizes retrieved content into a grounded, cited response. Runs behind generative orchestration, and as a configurable node inside topics.
Generative answers node
A topic node whose knowledge sources override the agent-level list — the only supported way to restrict a turn to specific sources. Instructions can't do this.
Generative orchestration
The current default runtime: it decides per turn which tools, knowledge, or topics to invoke, searches knowledge proactively, and unlocks the higher input counts (25 websites, 25 SharePoint URLs).
GraphSearch
SharePoint's live search infrastructure. The SharePoint connector path queries it in real time; content stays in SharePoint and nothing is copied or re-indexed.
Grounding
Building an answer from retrieved enterprise content, with citations, rather than from the model's general training. Validated with grounding, provenance, and semantic-similarity cross-checks.
H
Hybrid index
The combined lexical + semantic index that powers Microsoft 365 Copilot. The Retrieval API returns relevant text chunks directly from it, with no separate index to build.
I
Instructions
The agent's behavior surface: deciding which resources to call, filling tool inputs, and generating/shaping the response. They influence summarization after retrieval — never retrieval itself.
K
Knowledge source
Content attached to an agent that RAG retrieves from to ground answers. Supplies facts only: it cannot define behavior and is the wrong vehicle for examples or templates.
KQL (Keyword Query Language)
The query language for scoping M365 Copilot Retrieval API calls — filter by URL, date range, and file type.
L
Lexical retrieval
Keyword-only retrieval. In the Retrieval API it is all you get for file extensions outside .doc/.docx/.pptx/.pdf/.aspx/.one.
M
Managed property
A search-schema property whose content actually enters the SharePoint index. Its attributes — Searchable, Queryable, Retrievable — are the site owner's core control surface.
Moderation / RAI
Responsible-AI content moderation applied to incoming messages and generated responses (message moderation is step one of the website-grounding pipeline). High settings can filter an agent's own legitimate security-topic content.
O
OneDrive / SharePoint file sync
"Upload files from OneDrive/SharePoint": pulls specific files or folders into Dataverse with auto-sync and per-user permission trimming. Word, PowerPoint, PDF, and Excel only; about 5 items per add.
Orchestration
The runtime layer that plans each conversation turn — which topics, tools, and knowledge sources fire. Comes in generative (current default) and classic flavors.
Oversharing
Latent over-permissioning (e.g. "Everyone" has Read) that an agent makes suddenly visible and queryable. Permission trimming only enforces what your permissions actually say — audit before rollout.
P
Permission trimming
Retrieval-time filtering to content the querying user can read, via a live check at the source — no permission data is stored locally. No access means a silent "no response," not an error.
Proactive search
Generative orchestration's behavior of invoking knowledge search (via the Universal Search Tool) whenever it judges a turn needs grounding — not merely as a fallback when topics miss.
Public website (knowledge source)
Bing-grounded source restricted to your configured domains and two subpage levels of depth. Public, Bing-indexed pages only — auth-gated or unindexed pages ground poorly or not at all.
Q
Query filters (search query filters)
Include/exclude parameters — Title, Author, Modified by, Modified on — set on a SharePoint knowledge source. The highest-ROI Copilot-Studio-side lever for keeping stale content out of the top 3.
Query rewriting
Step one of the RAG pipeline: the user's raw message is rewritten into an optimized search query using conversation context (prior turns, location, time).
Queryable
Managed-property attribute enabling property-scoped queries (e.g. author:Smith). Note: Copilot Studio still won't do column-value filtering, even over queryable properties.
R
RAG (retrieval-augmented generation)
The four-step grounding pipeline: query rewrite → retrieve (top 3 per source) → summarize under your instructions → safety validation. Scoped to factual Q&A — not whole-document comparison or complex reasoning.
Re-crawl
The crawl pass required before any search-schema change (a new mapping, an attribute flip) takes effect. SharePoint Online gives you limited direct control over its timing.
Restricted SharePoint Search
Tenant setting that, when on, blocks the SharePoint connector path entirely.
Retrievable
Managed-property attribute that allows a value to be returned in search results. Disable it to hide indexed content from grounding.
Retrieval API (M365 Copilot)
A Preview developer API returning relevant text chunks from the hybrid index that powers M365 Copilot — KQL-scopable, permission-trimmed, and "keeps your data in place." A code path, distinct from Copilot Studio's built-in knowledge.
S
Search schema
The crawled→managed property mappings and attribute settings that govern what enters the SharePoint index and what search returns. The site owner's deepest lever; every change needs a re-crawl.
Searchable
Managed-property attribute that adds content to the full-text index so free-text (and semantic) queries can find it.
Semantic index
A vectorized index enabling similarity search "beyond exact keyword matching": items become vectors, and semantically similar items cluster together.
Sensitivity labels
Information-protection labels. The SharePoint connector respects them as a filter layer; on upload paths, labeled / encrypted / password-protected files won't index — they show "ready" but return nothing.
SharePoint connector (Path A)
The "Add knowledge → SharePoint" source: point at a URL and query GraphSearch live. Real-time, permission-trimmed, advanced filters, 25 URLs in generative mode. The default SharePoint RAG path.
Silent "no response"
What a user gets when permission trimming (or an empty result set) removes everything: no answer, no error, no exception. The number-one SharePoint troubleshooting trap.
Source filter (25-source rule)
In generative mode, when more than 25 sources are attached an internal GPT model filters which sources are considered per turn. Uploaded files are exempt from the count.
T
Tenant graph grounding (with semantic search)
The licensed setting that lets SharePoint retrieval ride the tenant semantic index: greater volume of context with greater precision, a small latency increase, and the 200 MB ceiling. Turn it on.
Tenant semantic index
The organization-wide semantic index Microsoft builds automatically from text-based SharePoint Online files for every paid-Copilot-license tenant. Not rebuildable or configurable — your lever is content and schema hygiene.
Third-party KB connectors
Confluence, Salesforce, ServiceNow, and Zendesk knowledge connectors: per-user credentials, published articles only, collection-level granularity (you can't pick individual articles).
Top-3 grounding
Only the top three search results per source are used to summarize each answer. The single most consequential retrieval constraint: precision beats volume, and near-duplicates crowd the right doc out.
Topics
Structured conversation flows within an agent — the home for deterministic behavior, and (via generative answers nodes) for scoping retrieval to specific sources.
U
Ungrounded responses
Answers generated from the model's general knowledge without retrieving from any source or tool. Permitted or blocked by the Allow ungrounded responses setting.
Universal Search Tool
The internal tool generative orchestration uses to invoke knowledge search proactively during a turn.
Uploaded files (documents source)
Files uploaded directly to the agent and stored/indexed in Dataverse. Widest format support (.md, .txt, .json…) and exempt from the 25-source filter — but static (manual re-upload) and visible to anyone with agent access.
W
Web Search ("Use information from the web")
A separate agent toggle that interleaves Bing web results with your knowledge sources' results. Distinct from the public-website knowledge source.
Where the definitions come from
